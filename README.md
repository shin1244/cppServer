# IOCP 기반 실시간 대전 게임 서버

Windows **IOCP**로 구현한 C++ 멀티스레드 게임 서버입니다. 실시간 top-down 슈팅 게임을 돌리면서, 서버 설계에서 마주치는 **핵심 선택 3가지를 각각 두 방식으로 직접 구현하고 실측 비교**하는 데 목적을 뒀습니다.

**기술 스택**: C++17 · Winsock2 IOCP · Go 클라이언트/부하봇

---

## 아키텍처

```
[IOCP 워커 ×32]                         [로직 스레드 ×1, 33ms 틱]
WSARecv ─▶ 세션 수신버퍼 ─▶ 프레임 파싱 ─▶ 큐 ─▶ 큐 소비 ─▶ World.Update ─▶ WSASend
          (링/가변)                    (더블스왑/이벤트)        (전체순회/그리드)
```

- 워커 스레드 32개가 패킷을 받아 파싱 후 스레드세이프 큐에 넣고, **단일 로직 스레드**가 33ms(~30Hz)마다 큐를 비워 처리합니다. 로직이 단일 스레드라 게임 상태에 락이 필요 없습니다.
- 방(World) 2개 · 방당 최대 200명 · 총알 4096 → 동시 400명 부하.
- 격자 맵에서 이동·사격, 총알로 벽 파괴, 피격 시 관전 전환, 시야(LOS)+반경 기반 AOI 전송.

---

## 핵심 실험 3가지

### 1. 세션 → 로직 스레드 전달 큐 — 더블 스왑 버퍼 vs 이벤트 큐
- **이벤트 큐**: 단일 큐 + 뮤텍스, push·pop 매 건마다 락
- **더블 스왑**: 버퍼 2개, 워커는 push 시만 락 / 로직은 틱당 1회 swap
- **결과**: 전체 틱 시간은 게임 로직 연산에 묻혀 차이가 안 보였음 → **순수 큐 소비 시간(Consume Time)** 으로 분리 측정하니 더블 스왑이 **약 2~3배 빠름**.
- **배운 점**: 락 경합의 이점은 상위 지표에 가려지므로 **측정 지점을 옳게 잡는 것**이 관건.

### 2. 세션 수신 버퍼 — 링 버퍼 vs 가변 버퍼
- **링 버퍼**: 고정 4096B, 메모리 상한 보장 / **가변 버퍼**: `vector`, 필요 시 grow
- **결과**: 정상 부하에선 소비가 생산을 따라잡아 **거의 동일**. 차이는 데이터가 쌓이는 backpressure 상황에서만 나타남.
- 부하 봇에 **악성 모드**(완성되지 않는 거대 패킷)를 추가해 검증 → 링은 상한 안에서 세션을 격리, 가변은 세션마다 메모리가 계속 증가(**DoS/OOM 위험**).
- **결론**: "어느 게 빠른가"가 아니라 **"최악의 경우 어떻게 무너지는가"** 가 진짜 차이. 링은 오버플로 정책(backpressure)이, 가변은 상한(cap)이 반드시 필요.
- 이 과정에서 링 버퍼의 full/empty 오인 버그와 backpressure 결함을 찾아 수정.

### 3. AOI 대상 선정 — 전체 순회 vs 공간 그리드
- **전체 순회**: 플레이어 O(N²) + 총알 O(N·M) / **그리드**: ≈ O(N·k)
- 후보 수집(broad-phase)만 교체하고 거리·시야 필터는 공유해 **두 방식의 결과를 동일**하게 맞춤 → 순수 알고리즘 비교.
- **핵심**: 그리드의 강점은 "빠름"보다 **N에 대한 확장성(제곱 → 선형)**. 유저가 분산될수록 격차가 커짐.

> 실험 2·3의 정량 수치는 측정 진행 중입니다.

## 빌드 & 실행

```bash
# 서버: cppServer.sln (Visual Studio / MSVC, x64) 빌드 후 실행 → 5050 포트 리슨

# 부하 봇 (Go)
cd client
go run ./bot -n 200                # 정상 봇 200개
go run ./bot -n 200 -mal 0.2       # 20% 악성 봇 (실험 2)

# 렌더링 클라이언트 (시야/전장의 안개 시각화)
go run .
```

**설계 토글**

| 실험 | 매크로 / 위치 | 정의 시 → 미정의 시 |
|---|---|---|
| 1 | `USE_EVENT_QUEUE` (`NetworkCore.h`) | 이벤트 큐 → 더블 스왑 |
| 3 | `USE_NOT_GRID` (`World.h`) | 전체 순회 → 그리드 |
| 2 | `Session`의 수신 버퍼 타입 (`NetworkCore.h`) | `RingBuffer` ↔ `VectorBuffer` |

## 디렉터리 구조

```
cppServer/
├─ cppServer/                # 서버 (C++)
│  ├─ main.cpp / main.h      # 진입점, 로직 루프, 벤치마크
│  ├─ NetworkCore.*          # IOCP 워커, 세션, 수신/송신
│  ├─ RingBuffer.* / VecterBuffer.*   # [실험2] 수신 버퍼
│  ├─ DoubleBuffer.h / EventQueue.h   # [실험1] 전달 큐
│  ├─ SpatialGrid.*          # [실험3] 공간 분할
│  ├─ World.* / World_Net.cpp# 게임 로직 / 패킷 핸들러
│  ├─ Player.* / Bullet.* / Map.*     # 게임 오브젝트
│  └─ Protocol.h / ObjectPool.h
└─ client/                   # 클라이언트 (Go)
   ├─ main.go                # 렌더링 클라 (ebiten)
   └─ bot/main.go            # 헤드리스 부하 봇 (+ 악성 모드)
```
