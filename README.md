## 1. 더블 스왑 큐 VS 이벤트 큐
네트워크 워커 스레드들이 수신한 패킷을 메인 로직 스레드로 전달하기 위한 Thread-Safe한 큐 구조가 필요했습니다.
GO언어에서 자주 사용하는 이벤트 큐 방식과 락 경합을 최소화 하기 위해 고안된 더블 스왑 방식을 구현하여 실측 성능 및 구조적 안정성을 비교, 분석해보았습니다.

### 이벤트 큐
작동 방식 단일 큐와 단일 뮤텍스를 공유합니다. 워커 스레드와 메인 로직 스레드는 패킷을 처리할 때마다 락을 잡습니다.
패킷이 대량으로 몰릴 경우, 워커 스레드와 메인 로직 스레드의 락 경합이 발생하여 메인 루프의 지연이 발생할 것으로 예상됩니다.

### 더블 스왑 큐
두개의 버퍼(read only/write only)를 두고 메인 로직 스레드는 

부하 테스트 시나리오
테스트 도구 go 언어로 제작한 헤드리스 부하 봇 고루틴으로 동시 접속
동시 접속자 수 400 한 방에 200명 씩
테스트 조건 워커 스레드 32개
초당 패킷 송신 10ms
